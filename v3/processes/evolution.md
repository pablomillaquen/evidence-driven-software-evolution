# Process — Evolution

**Weight**: Recommended

## Purpose

Implement changes to a system based on evidence and decisions.

## Flow

```
1. Review Findings
2. Define User Stories
3. Implement Changes
4. Validate with Regression
5. Record Evidence
6. Release
7. Update Baseline
```

## Entry Criteria

- Baseline frozen
- Findings documented
- Decisions accepted
- User Stories defined

## Exit Criteria

- Changes implemented
- Regression passed
- Evidence recorded
- Release tagged
- Baseline updated

## Uses

- Findings (from metamodel)
- Decisions
- User Stories
- Evidence
- Baselines

## Produces

- Implementation
- Regression evidence
- Updated Baseline
- Release
