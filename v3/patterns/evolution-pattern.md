# Pattern — Evolution

**Weight**: Recommended

## Context

System needs changes based on evidence and decisions.

## Problem

How to implement changes while preserving system integrity?

## Forces

- Changes must be evidence-based
- Regression must be avoided
- Architecture must be preserved
- Knowledge must be captured

## Solution

1. Review Findings
2. Define User Stories
3. Implement Changes
4. Validate with Regression
5. Record Evidence
6. Release
7. Update Baseline

## Expected Outputs

- Implementation
- Regression evidence
- Updated Baseline
- Release

## Known Variations

- Feature Evolution
- Refactoring Evolution
- Dependency Evolution
- Security Evolution

## Anti-patterns

- Implementing without evidence
- Skipping regression testing
- Not recording evidence
- Not updating baseline
