# Pattern — Health Report

**Weight**: Recommended

## Context

Technology health needs to be assessed and monitored.

## Problem

How to systematically assess and track technology health?

## Forces

- Multiple dimensions to assess
- Scores must be comparable over time
- Low scores must trigger action
- Must be reproducible

## Solution

1. Define Health Dimensions
2. Collect Metrics
3. Calculate Scores
4. Generate Recommendations
5. Identify Governance Triggers
6. Feed into Next SPEC

## Expected Outputs

- Health dimensions
- Scores per dimension
- Recommendations
- Governance feedback triggers

## Known Variations

- Technology Health
- Architecture Health
- Security Health
- Performance Health

## Anti-patterns

- Not measuring all dimensions
- Not tracking over time
- Ignoring low scores
- Not feeding into next SPEC
