# Evidence-Driven Software Evolution (v1.x)

> **Note**: This is the historical reference for EDSE v1.x. For current usage, see `../SKILL.md` and `../v2/`.

## Purpose

A methodology for evolving software systems with evidence, traceability, and architectural control. Designed for AI-assisted engineering projects where decisions must be justified, not assumed.

## Core Principles

1. Research Before Implementation
2. Evidence as Requirement
3. Architectural Guardrails
4. Measure First
5. Regression First
6. Complete Traceability
7. Reverse Traceability
8. Knowledge Management

## Evolution

EDSE v1.x was developed through:
- SPEC-001: Validation (85/85 tests)
- SPEC-002: Stabilization
- SPEC-003: Architectural Refactoring
- SPEC-004: Dependency Modernization
- SPEC-005: Frontend Assessment

The methodology evolved from a simple workflow to a layered architecture with its own conceptual language.

## Migration to v2.0

EDSE v2.0 reorganizes the methodology into:
- **Foundations**: Axioms that never change
- **Conceptual Model**: Entities and relationships
- **Principles**: Methodological rules
- **Processes**: Behavior patterns
- **Artifacts**: Documentary representations
- **Patterns**: Reusable scenarios
- **Templates**: Minimal structures

See `../v2/` for the current version.
