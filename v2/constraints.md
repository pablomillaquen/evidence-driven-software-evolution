# EDSE Constraints

**Weight**: Normative

## Purpose

Invariant requirements. EDSE implementations MUST comply with these rules.

## Evidence Constraints

- Every Finding MUST reference at least one Evidence
- Every Evidence MUST be traceable to its source
- Every claim MUST be backed by evidence

## Decision Constraints

- Every Decision MUST originate from a Candidate Decision
- Every Architectural Decision MUST have an ADR Document
- Every Decision MUST reference its justification

## Baseline Constraints

- Every Baseline MUST reference a frozen Evidence Package
- Every Baseline MUST include a snapshot
- Every Baseline MUST capture configuration state

## Traceability Constraints

- Every Finding MUST trace back to a Research Question
- Every Decision MUST trace forward to implementation or deferral
- Every Technical Debt MUST trace back to a Decision

## Artifact Constraints

- Every Artifact MUST be reproducible
- Every Artifact MUST include its evidence
- Every release MUST include a Release Report
