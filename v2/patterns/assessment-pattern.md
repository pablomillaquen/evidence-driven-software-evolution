# Pattern — Assessment

**Weight**: Recommended

## Context

System exists and needs evaluation without functional changes.

## Problem

How to systematically evaluate an existing system and produce actionable knowledge?

## Forces

- Limited time and resources
- Need for comprehensive understanding
- Requirement for evidence-based decisions
- Future evolution depends on current assessment

## Solution

1. Define Research Questions
2. Define Assessment Goals
3. Collect Evidence
4. Generate Findings
5. Prepare Candidate Decisions
6. Accept/Defer Decisions
7. Freeze Baseline
8. Generate Health Report
9. Governance Feedback

## Expected Outputs

- Evidence Package
- Findings with severity and confidence
- Candidate Decisions
- Frozen Baseline

## Known Variations

- Technology Assessment
- Architecture Assessment
- Security Assessment
- Performance Assessment

## Anti-patterns

- Mixing assessment with implementation
- Skipping Decision Preparation Phase
- Creating conclusions in Research Protocol
