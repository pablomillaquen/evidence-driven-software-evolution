# Process — Governance

**Weight**: Recommended

## Purpose

Guide software evolution through evidence-based decisions.

## Flow

```
1. Review Health Report
2. Identify Low Dimensions
3. Generate Research Questions
4. Feed into Next SPEC
5. Update Roadmap
```

## Entry Criteria

- Health Report available
- Baseline frozen

## Exit Criteria

- Research Questions generated
- Roadmap updated
- Next SPEC identified

## Governance Feedback

When a dimension scores below 70, it automatically generates Research Questions for the next SPEC.

## Health Dimensions

| Dimension | Score Threshold |
|-----------|-----------------|
| Dependency Health | < 70 triggers feedback |
| Security Posture | < 70 triggers feedback |
| Code Quality | < 70 triggers feedback |
| Testing Maturity | < 70 triggers feedback |
| Documentation Coverage | < 70 triggers feedback |
