# Process — Assessment

**Weight**: Recommended

## Purpose

Systematically evaluate an existing system and produce actionable knowledge.

## Flow

```
1. Identify Research Questions
2. Define Assessment Goals
3. Collect Evidence
4. Generate Findings
5. Prepare Candidate Decisions
6. Accept/Defer Decisions
7. Freeze Baseline
8. Generate Health Report
9. Governance Feedback
```

## Entry Criteria

- System exists
- Access to codebase
- Testing infrastructure

## Exit Criteria

- Evidence Package complete
- Findings documented
- Candidate Decisions prepared
- Baseline frozen

## Uses

- Research Questions (from metamodel)
- Assessment Goals
- Evidence
- Findings
- Candidate Decisions
- Decisions
- Baselines
- Health Reports

## Produces

- Evidence Package
- Findings with severity and confidence
- Candidate Decisions
- Frozen Baseline
