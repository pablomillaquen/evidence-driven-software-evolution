# Process — Baseline

**Weight**: Recommended

## Purpose

Capture system state at a specific point in time.

## Flow

```
1. Capture Current State
2. Validate Evidence Packages
3. Accept/Defer Candidate Decisions
4. Create Snapshot
5. Freeze Configuration
6. Reference Point Established
```

## Entry Criteria

- Evidence Package complete
- Findings documented
- Candidate Decisions prepared

## Exit Criteria

- Snapshot created
- Configuration captured
- Reference point established

## Freeze Criteria

Before freezing, verify:
- [ ] All evidence packages are complete
- [ ] All findings are documented
- [ ] All candidate decisions are accepted or deferred
- [ ] Baseline snapshot is created
- [ ] Configuration state is captured

## What is Frozen

- Evidence Package
- Findings with severity and confidence
- Candidate Decisions (accepted or deferred)
- Technology Health Report
- Baseline snapshot

## What is NOT Frozen

- Permanent documentation (separate deliverable)
- Implementation decisions (future SPECs)
- New features (future SPECs)
