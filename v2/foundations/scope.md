# EDSE — Scope

## When to Use

- Evolving an existing software system with controlled changes
- Stabilizing a codebase before adding new features
- Projects requiring audit trails and architectural decisions
- Any project where "why" matters as much as "what"

## What EDSE Governs

- Investigation methodology
- Evidence collection and organization
- Decision-making processes
- Baseline management
- Technology health assessment
- Knowledge management

## What EDSE Does NOT Govern

- Implementation details
- Programming languages
- Framework choices
- Team organization
- Project management

## Applicability

EDSE is technology-agnostic. It can be applied to:

- Backend systems (any language/framework)
- Frontend applications (any framework)
- Mobile applications
- Legacy systems
- Greenfield projects
- Migrations
