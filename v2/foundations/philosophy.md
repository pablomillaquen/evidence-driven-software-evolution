# EDSE — Philosophy

## Identity

Evidence-Driven Software Evolution (EDSE) is a software engineering methodology built upon a conceptual language for describing and governing software evolution.

## Core Belief

The true deliverable is not individual documents but the complete Evidence Package. Documents merely contain that evidence.

## Paradigm

EDSE is not oriented to executing changes. It is oriented to building reliable knowledge to govern software evolution.

```
Research Question
        ↓
    Evidence
        ↓
    Finding
        ↓
    Decision
        ↓
   Knowledge
```

## Evidence Philosophy

Every important claim must be backed by evidence. Not opinion. Not intuition. Not "it feels faster."

The answer to "why did you modernize the frontend?" is not "because the code was old" — it is the full chain: RQ → Evidence → Finding → Candidate Decision → Accepted Decision → Future SPEC.

## Methodology + Language

EDSE is a methodology that possesses its own conceptual language:

- **Conceptual Model** (entities and relationships)
- **Principles** (methodological guidelines)
- **Processes** (behavior patterns)
- **Artifacts** (documentary representations)
- **Patterns** (reusable scenarios)

Like DDD (Domain-Driven Design), EDSE is a methodology that also possesses a language.
